# 整理券アプリ（順番待ち発券機）MVP 設計

作成日: 2026-06-01

## 目的

飲食店・病院・役所などで使う「順番待ちの整理券システム」のMVP（最小限の動く試作品）を作る。お客様が番号を取り、スタッフが順番に呼び出し、店頭ディスプレイに現在の呼び出し番号を表示する。

## 方針

- 単体HTMLファイル（HTML + CSS + JavaScript）で実装する。
- データはブラウザ内の localStorage に保存する。サーバーは使わない。
- ダブルクリックでブラウザで開くだけで動く。インストール不要。
- 同じブラウザで複数タブを開いた場合、storage イベントで自動連動させる（管理画面で呼ぶと表示画面が更新される）。

## 全体像

1つのHTMLファイルの中に「3つの画面（モード）」を持たせ、画面上部の切り替えで表示を変える。3画面は同じ localStorage を共有する。

## 画面仕様

### 1. 受付画面（お客様用）
- 大きな「整理券を取る」ボタン。
- 押すと最新番号を +1 して発行し、「あなたの番号は NN です」と大きく表示する。
- 現在の待ち人数を表示する。

### 2. 呼び出し管理画面（スタッフ用）
- 待っている番号の一覧を表示する。
- 「次を呼ぶ」ボタン: 待ちリストの先頭を取り出し、現在の呼び出し番号にする。
- 「リセット」ボタン: すべての番号を初期状態（0）に戻す。確認ダイアログを挟む。

### 3. 呼び出し表示画面（店頭ディスプレイ用）
- 「ただいまの呼び出し番号　**NN**」を画面いっぱいに大きく表示する。
- 管理画面で番号が進むと自動で更新される（storage イベント連動）。

## データモデル（localStorage）

- `lastIssued`: 発行済みの最新番号（整数。初期値 0）
- `nowCalling`: 現在の呼び出し番号（整数。初期値 0、未呼び出しは 0 または「---」表示）
- `waiting`: 待ち番号のリスト（整数の配列）

発券時: `lastIssued += 1`、`waiting` に push。
呼び出し時: `waiting` の先頭を shift し `nowCalling` に設定。
リセット時: 3つすべてを初期値に戻す。

## エラー・端の処理

- 待ちリストが空のときに「次を呼ぶ」を押したら、何もせず「待っている番号はありません」と表示する。
- localStorage に値が無い初回起動時は、すべて初期値で開始する。

## MVPに含めないもの（将来の拡張）

- スマホからのQRコード取得・スマホ通知
- 複数端末をまたいだリアルタイム同期（サーバーが必要）
- 整理券の印刷、音声呼び出し、待ち時間予測
- 複数の窓口・カテゴリ分け

## 動作確認

- 受付画面で複数回発券し、番号が連番で増えることを確認する。
- 管理画面で「次を呼ぶ」を押すと、呼び出し番号が待ち順に進むことを確認する。
- 表示画面を別タブで開き、管理画面で呼ぶと自動更新されることを確認する。
- リセットで初期状態に戻ることを確認する。
