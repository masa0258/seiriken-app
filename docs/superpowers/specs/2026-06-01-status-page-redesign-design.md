# 呼び出し状況ページ（status.html）リデザイン 設計

作成日: 2026-06-01

## 目的

顧客向け呼び出し状況ページ `status.html` の見た目を刷新し、「AIで作った感・素人感」を払拭して、カフェ・飲食店にふさわしい洗練された印象に整える。機能・ロジックは一切変えず、CSSとHTML構造だけで上質さを出す。

## コンセプト

**温かみ・ホスピタリティ × テラコッタ。** クリーム色のやわらかい地に白カードを浮かせ、テラコッタ（サンセット系オレンジ）をアクセントにする。親しみやすく、安心感があり、上品。

## スコープ

### 今回やること
- `status.html` の `<style>` ブロックを全面刷新。
- `status.html` の `<body>` のHTML構造を整える（クラス付与、カード化）。
- 「直近の呼び出し」を文字列連結から**チップ（丸枠タグ）表示**へ変更。これに伴い `renderRow` 内の `recent-called` 描画だけを「番号ごとに `<span>` を生成して append」する最小JS変更を行う（`innerHTML` は使わない）。
- 全画面通知オーバーレイ（`showOverlay`）の見た目をテラコッタ基調に合わせる（CSSクラスのみ。DOM生成方法は現状維持）。

### 今回やらないこと（YAGNI）
- 表示する情報・項目の増減（番号・お待ち組数・直近呼び出し・通知・注意書きの構成は維持）。
- 通知ロジック・ポーリング・localStorage・Supabase連携の変更。
- `index.html`（店アプリ）・`status.js`・`queue.js`・Supabase の変更。
- 外部Webフォント（Google Fonts等）の読み込み。自己完結・高速表示・追加通信ゼロを優先し、端末標準フォントを使う。
- ダークモード対応・多言語化・テーマ切り替え。

## 確定した方針

- **方向性**: 温かみ・ホスピタリティ（モダンミニマル／ダークヒーローではない）。
- **配色**: テラコッタ（キャラメル／ローズではない）。
- **フォント**: 端末標準の日本語フォント（iOS=ヒラギノ角ゴ、Android/Windows=游ゴシック等）。外部フォント不使用。
- **変更範囲**: `status.html` のみ（CSS刷新＋HTML微修正＋`recent-called` 描画の最小JS変更）。
- **ロジック不変**: 既存の要素IDとJSの振る舞いは維持。`status.js` の30件テストがPASSし続けることで担保。

## デザイン仕様

### カラーパレット
- 背景: `#fbf4ec`（クリーム）
- ヒーローグラデ: `#ff8a5b` → `#ef6c47`（160deg）
- アクセント（ボタン・数字・チップ文字）: `#ef6c47`
- アクセント濃（通知文字など）: `#c2410c`
- 通知カード地: `#fff3e8`
- チップ地: `#fde7da` / チップ文字: `#c2410c`
- カード地: `#ffffff`
- 基本文字色: `#4a3b2e` / 店名: `#9a6b3f` / 補助文字: `#9b8a78`〜`#a89684`
- 入力枠: `#ecd9c5` / 入力プレースホルダ文字: `#b08968`

### 形・影
- カード角丸: 16px（ヒーローは22px、入力・ボタンは14px、チップは999px=ピル）
- 影: `0 2px 8px rgba(154,107,63,.10)`（カード）、`0 6px 16px rgba(239,108,71,.35)`（ヒーロー）
- 全体の最大幅: 既存どおりモバイル前提（`max-width: 480px` 程度）、中央寄せ

### タイポ
- フォントスタック: `'Hiragino Sans','Hiragino Kaku Gothic ProN','Yu Gothic','Meiryo',system-ui,sans-serif`
- 呼び出し番号: 約60px、太字（800）、白文字
- お待ち組数の数字: 約20px、太字、アクセント色
- 店名: 14px、太字、店名色

### レイアウト（上から）
1. **店名** — 中央、店名色、控えめ。
2. **呼び出し番号ヒーロー** — テラコッタのグラデカード、白文字。ラベル「ただいまお呼び出し中」＋大きな番号＋「番」。番号が無い時は「準備中です」を同カード内に白抜きで表示（赤の警告色は使わない）。
3. **あなたの番号 確認** — 白カード。入力欄（プレースホルダ「あなたの番号」）＋「確認」ボタン（テラコッタ）。結果テキスト（`my-result`）はその下に表示。
4. **通知ボタン** — 淡いオレンジ地のカード調。未登録「🔔 通知を受け取る」、登録中「🔔 通知ON（◯番）」＋「解除」。`notify-status` も同カード内。
5. **お待ち組数** — 白カード。「現在 N 組お待ち」、数字だけアクセント色で大きく。
6. **直近の呼び出し** — 白カード。ラベル「直近の呼び出し」＋番号をチップ（ピル）で横並び。空の時は「—」。
7. **注意書き** — 最下部に小さく「※通知のため、この画面を開いたままにしてください」。
8. **メタ** — 「最終更新 …（自動更新中）」。`再接続中…` 時は控えめな警告色（既存の `.err` 相当、ただしテラコッタ調の落ち着いた色）。

### 全画面通知オーバーレイ
- 背景: 半透明の暗幕（`rgba(0,0,0,.55)` 程度）。
- 中央カード: 白、角丸、影。`turn`（あなたの番です！）はテラコッタ系の強いトーン、`soon`（まもなくお呼び出しです）は淡いオレンジの優しいトーンでタイトル色を出し分け。
- 内容: タイトル＋大きな番号（`◯番`）＋「閉じる」ボタン。DOM生成は現状維持（`createElement`/`textContent`/`appendChild`、`innerHTML` 不使用）。

## 維持する技術的制約（重要）

- **要素IDを変えない**: `store-name`, `calling-number`, `my-number`, `my-check`, `my-result`, `notify-btn`, `notify-status`, `waiting-count`, `recent-called`, `meta`。JSがこれらを参照しているため。
- **JSの振る舞いを変えない**: `poll`/`renderRow`/`renderMyResult`/`maybeNotify`/`evaluateNotify`/通知ラッチ/localStorage は不変。例外は `recent-called` のチップ描画（表示方法のみ変更、対象データは不変）。
- **innerHTML 不使用**を維持。
- 外部リソース（フォント・画像・CSS/JSライブラリ）を新たに読み込まない。

## データフロー

変更なし。`poll()` が5秒ごとにSupabaseから行を取得 → `renderRow()` が各要素を更新 → 通知判定 `maybeNotify()`。リデザインは描画先のDOM構造とスタイルのみを変える。唯一 `recent-called` の描画が「文字列代入」から「チップ要素の生成・追加」に変わる。

## エラー・端の処理（既存維持）

- 通信失敗 → 「再接続中…」、前回値保持（メタの警告色表示）。
- 呼び出し番号 null → ヒーロー内「準備中です」。
- 直近呼び出しが空 → 「—」。
- 自分の番号が待機に無い → 既存の結果文言を維持。

## テスト

- **自動**: 既存 `status.test.html` の30件（`vm` ハーネス）が引き続き PASS すること。これによりロジック無変更を担保する。
- **静的**: `grep -c innerHTML status.html` が 0。
- **手動（見た目）**: スマホ実機で各カード・ヒーロー・チップ・通知ボタン・全画面オーバーレイ（soon/turn 両方）の表示を目視確認。準備中・再接続中の表示も確認。

## 手動検証

1. ローカルまたは公開URLの `status.html` をスマホで開き、新デザインが表示される。
2. 番号入力・確認、通知登録/解除でレイアウトが崩れない。
3. 店アプリで番号を進め、「まもなく」「あなたの番です！」オーバーレイがテラコッタ基調で表示される。
4. 準備中（番号なし）・再接続中（通信断）の表示が自然。
5. `status.js` テスト30件PASS、`innerHTML` 0 を確認。
